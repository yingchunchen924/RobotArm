# 操作手册 —— 电力库房机械臂系统真机部署

> 给团队成员：本手册带你从拿到压缩包，一路做到在 Atlas 200I DK A2 开发板上跑通
> 完整系统（识别 → 抓取 → 分类 → 可视化）。**请按顺序做，不要跳步。**
>
> 阅读约定：
> - 🖥️ = 在你自己的电脑（PC）上做
> - 🤖 = 在 Atlas 200I DK A2 开发板上做
> - ⚠️ = 容易踩坑的地方

---

## 0. 先搞清楚这个项目是什么

这是一套"摄像头识别物品 → 机械臂抓取 → 按类别分拣/堆叠 → 网页实时显示"的系统。

代码分三层，**理解这点能少走很多弯路**：

| 层 | 在哪 | 能在 PC 跑吗 | 说明 |
| --- | --- | --- | --- |
| 纯逻辑层 | `src/robotarm/` | ✅ 能 | 坐标转换、目标筛选、抓取流程、库存。不碰硬件，已写好且测试通过 |
| 真机层 | `ros2_ws/` | ❌ 不能 | ROS2 逆解、NPU 推理、机械臂串口。**需要你在开发板上补实现** |
| 可视化层 | `web/` | ✅ 能 | 网页后端+前端，mock 模式 PC 上就能看 |

**核心思路**：纯逻辑层已经在 PC 上验证好了（65 项测试全绿），你在真机上要做的，**只是把几个"接口"接到真实硬件**，上层逻辑一行都不用改。具体接哪几个，见第 5 节。

## 1. 🖥️ 在 PC 上先把它跑起来（10 分钟，强烈建议先做）

先在自己电脑上跑通"假硬件"版本，理解系统怎么转，再上真机就有底了。

```bash
# 进入项目目录
cd RobotArm

# 安装依赖（Python 3.8+）
pip install -r requirements.txt

# 1) 跑测试，确认环境没问题（应显示 65 passed）
pytest tests/ -q

# 2) 命令行看完整流程：分拣 + 堆叠 + 电力物品分类
python scripts/demo_full_system.py

# 3) 看网页界面（mock 模式，硬件是假的，但状态/库存全真实流转）
bash scripts/start_demo.sh
# 浏览器打开 http://127.0.0.1:8000/ ，点「开始识别抓取」，看状态和库存变化
```

⚠️ 如果 `pip install` 后 `import onnxruntime` 报 DLL 错误，把版本降一下：
`pip install onnxruntime==1.16.3`（这是某些 Windows 缺新版 VC++ 运行库导致的）。
onnxruntime 只影响 PC 上的识别验证，不装也不影响其它功能。

> 跑通这步后，你就知道"正常情况下系统长什么样"。真机上出问题时好对照。

## 2. 🤖 开发板硬件连接与系统准备

照《05机械臂使用手册.pdf》第 2 章 + `docs/环境部署记录.md` 做：

1. PC、路由器、开发板接同一网络（开发板 eth0 网口接路由器）。
2. SD 卡烧录 **Ubuntu OS** 镜像（手册推荐，openEuler 未验证）。
3. 40PIN 排线连接开发板与机械臂扩展板 —— ⚠️ **接对时扩展板蓝灯亮**，不亮就是接反/没插好。
4. USB 摄像头插开发板，固定在机械臂上。
5. 开发板、机械臂分别通电，扩展板开关拨到 **ON**。
6. 用 MobaXterm 远程登录开发板（root / 默认密码 `Mind@123`），IP 在路由器后台查。

把过程记录到 `docs/环境部署记录.md`（有现成模板，填空即可）。

## 3. 🤖 开发板环境配置

⚠️ **每开一个新终端，都要重新 `source setenv.sh` 配环境变量**，否则 `ros2` 命令找不到。

```bash
# 1. 下载华为官方样例代码（提供 ROS2 功能包、运动学库、.om 样例模型）
cd /usr/local
bash E2E_samples_download_tool.sh -d /home/HwHiAiUser/E2ESamples \
  -s https://gitee.com/HUAWEI-ASCEND/ascend-devkit.git \
  -b master src/E2E-Sample/ros2_robot_arm

# 2. 按 scripts/setup_env.sh 里的步骤装依赖（ROS2 Humble / ais_bench / 运动学库 / 驱动）
#    ⚠️ 这个脚本是“步骤清单”，建议对照着一条条执行并确认每步成功，不要无脑整脚本跑
bash scripts/setup_env.sh   # 或逐条手动执行

# 关键检查点：
#   - libdofbot_kinemarics.so 已 cp 到 /usr/lib
#   - LD_LIBRARY_PATH 已加到 ~/.bashrc
#   - ros2_ws 已 colcon build 成功
```

⚠️ 常见坑：
- **`conda` 干扰**：手册要求在 `~/.bashrc` 末尾加 `conda deactivate`，不然 Python 环境会乱。不用机械臂时记得注释掉。
- **`colcon build` 失败**：多半是 ROS2 没装全或没 `source` 环境。先 `source /opt/ros/humble/setup.bash`。

## 4. 🤖 相机标定（首次必做）

机械臂能不能抓准，全看这一步。照手册 3.3 节做：

```bash
# 新终端起 jupyter
jupyter notebook --allow-root <开发板IP>
```

1. 打开样例里的 `相机校准.ipynb`，Kernel → Restart & Run All。
2. 点 `calibration_model`，调 `joint1`/`joint2`，让**蓝色边框盖住整个十字标定框**（光线要足）。
3. 点 `calibration_ok`，生成 `dp.bin` / `XYT_config.txt` / `offset.txt`。
4. 把这三个文件放到本项目 `config/` 目录下。

⚠️ 抓取有偏差时调 `offset.txt`（详见 `config/calibration.md`）：
- 抓得**偏后**（落在物体后面）→ **加大**参数（如 0.008→0.01）
- 抓得**偏前** → **减小**参数（如 0.008→0.006）
- 一次只改一个参数，改完测一次，别一次改多个。

## 5. 🤖 接真机：补 3 个接口实现（本项目的核心工作）

PC 上识别/逆解/机械臂用的是"假的"（Mock）。真机上你要实现 3 个真的，**接口已经定义好，照着填**。

### 5.1 机械臂串口驱动 `SerialArmDriver`
新建 `ros2_ws/src/power_arm_control/power_arm_control/serial_arm.py`，
继承 `robotarm.interfaces.ArmDriver`，实现 3 个方法（调华为样例的串口接口）：

```python
from robotarm.interfaces import ArmDriver
# 华为样例提供的机械臂对象，假设叫 Arm_Lib.Arm_Device()

class SerialArmDriver(ArmDriver):
    def __init__(self):
        import Arm_Lib
        self.arm = Arm_Lib.Arm_Device()
    def move_joints(self, joints, duration_ms=1000):
        self.arm.Arm_serial_servo_write6_array(list(joints), duration_ms)
    def set_gripper(self, angle, duration_ms=500):
        self.arm.Arm_serial_servo_write(6, angle, duration_ms)
    def home(self):
        self.move_joints([90, 130, 0, 0, 90, 30], 1000)
```

### 5.2 逆运动学客户端 `Ros2Kinematics`
新建 `ros2_ws/src/power_arm_control/power_arm_control/ros2_kinematics.py`，
继承 `robotarm.interfaces.Kinematics`，在 `inverse(x, y)` 里调 ROS2 `Kinemarics` 服务
的 `ik` 模式（参考手册 4.5 的 `server_joint`，逻辑可直接抄）：

```python
from robotarm.interfaces import Kinematics

class Ros2Kinematics(Kinematics):
    def __init__(self):
        # 初始化 ROS2 node + client（参考手册 server_joint）
        ...
    def inverse(self, x, y, z=0.0):
        # 发请求 tar_x=x, tar_y=y, kin_name="ik"，返回 [joint1..joint5]
        ...
```

> 逆解越界修正（手册里 `if joints[2]<0...` 那段）**不用你写**，本项目的
> `grasp_motion.fix_ik_overflow` 已经做了。你只要返回原始逆解结果即可。

### 5.3 识别模型（不用写代码，只需准备 .om 文件）
`AclDetector` 已写好，你只要把训练好的 `.om` 模型放到 `models/power_objects.om`。
怎么得到这个模型见第 6 节。

✅ 这 3 个补完，真机就能跑。**上层 `pipeline.py` / `runner.py` / `web/` 完全不用动。**

## 6. 🖥️🤖 准备识别模型（数据集 → 训练 → 转换）

这步要采集真实电力物品图片。完整流程在 `dataset/README.md`，简述：

```bash
# 🖥️ 1. 采集图片（需摄像头 + 实物，每类 100~300 张）
python scripts/dataset/collect_images.py --class resistor
python scripts/dataset/collect_images.py --class wrench --auto --interval 0.5 --count 200

# 用 LabelImg / Roboflow / CVAT 标注，导出 YOLO 格式到 dataset/labeled/

# 🖥️ 2. 校验 + 划分 + 生成配置
python scripts/dataset/validate_labels.py --images dataset/labeled/images --labels dataset/labeled/labels
python scripts/dataset/split_dataset.py --images dataset/labeled/images --labels dataset/labeled/labels --out dataset/yolo
python scripts/dataset/gen_dataset_yaml.py --root dataset/yolo --out dataset/yaml/power_objects.yaml

# 🖥️ 3. 训练（需 GPU，需先 git clone yolov5）
python scripts/model/train_yolov5.py --yolov5 /path/to/yolov5 --data dataset/yaml/power_objects.yaml

# 🖥️ 4. 导出 ONNX（见 scripts/model/export_onnx.md）
# 🤖 5. 在开发板用 ATC 转 .om（见 scripts/model/convert_om.sh）
#       bash scripts/model/convert_om.sh power_objects.onnx power_objects
#       产出 power_objects.om 放到 models/
```

⚠️ **类别千万别在数据集脚本里手写**。所有类别由 `config/categories.yaml` 统一管理，
脚本自动读取。要改识别类别，**只改 `categories.yaml` 一处**，然后重新生成 yaml。
否则会出现"数据集类别和系统类别对不上"的诡异问题。

⚠️ 时间紧可先少做几类（3~5 类），把流程跑通再加。

## 7. 🤖 一键启动整个系统

3 个接口补完、模型就位、标定完成后：

```bash
bash scripts/start_all.sh
# 它会按顺序起：ROS2 逆解服务 → 识别抓取主程序 → Web 后端
# 浏览器打开 http://<开发板IP>:8000/
```

⚠️ 启动顺序不能乱（脚本已处理好）：逆解服务必须先起，识别主程序才能调它。
日志在 `logs/` 目录，出问题先看那里。

也可以单独跑识别抓取主程序（不起 Web）：
```bash
python scripts/run_power_sorting.py --backend acl --model models/power_objects.om
```

## 9. 🚑 常见问题速查

| 现象 | 原因 | 解决 |
| --- | --- | --- |
| `ros2: command not found` | 没配环境 | `source setenv.sh`（每个新终端都要） |
| `colcon build` 失败 | ROS2 没装全 | `source /opt/ros/humble/setup.bash` 后重试 |
| 扩展板蓝灯不亮 | 40PIN 排线接反/松 | 重新插紧，对照手册图 2-3 |
| 识别主程序报"无法加载逆解客户端" | 逆解服务没起 / `Ros2Kinematics` 没实现 | 确认 ROS2 服务先起；检查 `logs/ros2_server.log` |
| 识别主程序报"无法加载串口驱动" | `SerialArmDriver` 没实现 | 见第 5.1 节；或先 `--mock` 烟测 |
| 机械臂抓偏 | 标定不准 / offset 没调 | 重新标定；调 `config/offset.txt`（第 4 节） |
| 逆解失败、机械臂不动 | 目标超出可达范围 | 调 `config/arm.yaml` 的 `reach_bounds` |
| 夹爪抓不稳、物体滑落 | 夹爪角/下降高度不对 | 调 `config/grasp.yaml` 对应类别参数 |
| 识别框乱跳 | 光照变化 | 固定光源；数据集补不同光照的图 |
| 网页状态不更新 | 后台没启动任务 | 点「开始识别抓取」；检查 `logs/web.log` |
| 网页端口被占用 | 8000 被占 | `bash scripts/start_all.sh 8001` 换端口 |
| Python 包冲突/找不到 | conda 干扰 | `conda deactivate`；确认用系统 Python |
| `import onnxruntime` DLL 错误（PC） | VC++ 运行库旧 | `pip install onnxruntime==1.16.3` |

## 10. 改东西去哪改（配置速查）

**所有可调参数都在 `config/` 里，改配置不用改代码：**

| 想改什么 | 改哪个文件 |
| --- | --- |
| 机械臂姿态/夹爪角/速度/坐标转换参数/可达范围 | `config/arm.yaml` |
| 各物体的夹爪夹紧角、放到哪个库区的姿态 | `config/grasp.yaml` |
| 色块堆叠每一层的位置 | `config/stacking.yaml` |
| 识别类别、类别→库区映射、中文名 | `config/categories.yaml` |
| 网页端口、视频源、mock 开关 | `config/web.yaml` |
| 标定文件路径 | `config/arm.yaml` 的 `calibration_files` |

⚠️ 调参铁律：**一次只改一个变量，改完测一次**。同时改多个出了问题不知道是哪个。
