# RobotArm 最新操作手册

本文档适用于本项目当前版本：电力库房智能识别与机械臂抓取系统。系统基于华为 Atlas 200I DK A2 智能机械臂，包含颜色方块抓取/堆叠、电力物品分类抓取、Web 可视化管理和 PC 端 Mock 演示。

## 1. 项目结构

```text
RobotArm/
├── config/          # 机械臂、抓取、堆叠、类别映射、Web 配置
├── dataset/         # 数据集占位与说明
├── docs/            # 开发计划、部署记录、测试记录、联调文档、前端手册
├── models/          # 模型文件占位，真机部署时放 .om 模型
├── photos/          # 演示或采集图片
├── ros2_ws/         # Atlas 开发板上的 ROS2 功能包
├── scripts/         # 环境配置、演示、数据集、模型和板端脚本
├── src/robotarm/    # 纯 Python 逻辑：坐标、筛选、库存、状态机、流程编排
├── tests/           # 单元测试和集成测试
└── web/             # FastAPI 后端与前端页面
```

核心分层：

| 层级 | 运行位置 | 作用 |
| --- | --- | --- |
| 纯逻辑层 `src/robotarm/` | Windows/Linux PC | 坐标转换、目标筛选、流程编排、库存统计，可直接测试 |
| 真机层 `ros2_ws/` 和 `scripts/board/` | Atlas 开发板 | ROS2 逆解、相机、机械臂串口、NPU 推理 |
| 可视化层 `web/` | PC 或 Atlas 开发板 | 浏览器控制、状态展示、库存统计、操作日志 |

## 2. PC 本机快速演示

PC 演示不需要真实机械臂，硬件由 Mock 接口模拟，适合答辩前预演和代码验证。

### 2.1 安装依赖

```bash
cd RobotArm
pip install -r requirements.txt
```

`requirements.txt` 主要包含：

- `pyyaml`
- `numpy`
- `opencv-python`
- `pytest`
- `fastapi`
- `uvicorn`
- `onnxruntime` 和 `onnx`，用于 PC 端模型验证

### 2.2 运行测试

```bash
pytest tests/ -q
```

测试覆盖配置加载、坐标转换、目标筛选、抓取流程、检测后处理、Runner 和 Web 接口。测试通过后，说明 PC 可验证逻辑正常。

### 2.3 命令行端到端演示

```bash
python scripts/demo_full_system.py
```

该脚本会串联：

1. 颜色分拣流程
2. 色块堆叠流程
3. 电力物品分类抓取流程
4. 库存统计和操作日志

### 2.4 PC Web 演示

```bash
bash scripts/start_demo.sh
```

然后浏览器打开：

```text
http://127.0.0.1:8000/
```

PC Web 演示使用 Mock 模式，可以验证页面按钮、状态流转、库存统计和日志记录。

## 3. Atlas 开发板真机运行

真机运行前必须确认硬件连接和环境都已经准备好。

### 3.1 前置条件

1. Atlas 200I DK A2 已开机并可通过 SSH/MobaXterm 登录。
2. 机械臂扩展板已通电，舵机和夹爪连接正常。
3. 摄像头连接到开发板，通常为 `/dev/video0`。
4. ROS2、昇腾运行环境、ais_bench 等已按 `scripts/setup_env.sh` 配置。
5. 标定文件已经生成并放入 `config/`，包括 `dp.bin`、`XYT_config.txt`、`offset.txt`。
6. 模型文件已经转换为 `.om` 并放入 `models/`。
7. 真机串口驱动和 ROS2 逆解服务已经实现并可运行。

### 3.2 一键启动完整系统

在开发板项目目录下执行：

```bash
cd /home/HwHiAiUser/RobotArm
bash scripts/start_all.sh
```

`start_all.sh` 会按顺序启动：

1. ROS2 运动学服务 `dofbot_server`
2. 识别抓取主程序 `run_power_sorting.py`
3. Web 后端 `uvicorn app:app`

启动后浏览器访问：

```text
http://<开发板IP>:8000/
```

如果使用常见直连地址，可尝试：

```text
http://192.168.137.100:8000/
```

### 3.3 分开启动检查

如果一键启动失败，可分开检查。

检查 ROS2 服务：

```bash
ros2 service list | grep trial_service
```

启动 Web 后端：

```bash
cd /home/HwHiAiUser/RobotArm/web/backend
uvicorn app:app --host 0.0.0.0 --port 8000
```

运行电力物品分拣：

```bash
cd /home/HwHiAiUser/RobotArm
python3 scripts/run_power_sorting.py
```

## 4. Web 页面操作

Web 页面主要包含实时画面、机械臂控制、实物分类、识别结果、库存统计和操作日志。

### 4.1 颜色方块抓取

适用于红、黄、蓝、绿方块。

操作步骤：

1. 把方块放到摄像头可见区域。
2. 打开 Web 页面，确认实时画面正常。
3. 点击单色抓取按钮，例如抓红色、抓黄色、抓蓝色、抓绿色。
4. 页面会显示当前流程，机械臂完成抓取后库存统计更新。

也可以使用连续抓取：

1. 按顺序选择红、黄、蓝、绿。
2. 点击按顺序抓取或连贯全抓。
3. 等待流程完成后检查操作日志。

### 4.2 色块堆叠

堆叠流程用于把多个色块码放到指定位置。

操作步骤：

1. 将色块摆放到取料区。
2. 点击堆叠按钮。
3. 系统按配置中的层位执行抓取和放置。
4. 如果已堆好的方块被识别成新目标，需要在脚本参数中设置排除区域，避免重复抓取。

命令行堆叠示例：

```bash
python3 scripts/board/color_sort_ros.py stack --color any --layers 4 --execute
```

### 4.3 电力物品分类抓取

适用于铜端子、铜管、U 盘、电池等真实物体。不同类别会放入不同库区。

网页操作流程：

1. 点击拍工作台照片。
2. 选择物体类别。
3. 在照片上点击物体中心位置。
4. 单次抓取会立即执行；队列模式会先加入队列，点击执行队列后统一运行。
5. 抓取完成后查看库存统计和操作日志。

点击位置建议：

- 点击物体中心或夹爪容易夹住的位置。
- 避开孔洞、边缘、反光严重的位置。
- 物体之间留出夹爪下降空间。
- 每次抓取后重新拍照，避免物体被碰动导致坐标不准。

## 6. 配置文件说明

| 文件 | 说明 |
| --- | --- |
| `config/arm.yaml` | 机械臂姿态、速度、夹爪、坐标转换和可达范围 |
| `config/grasp.yaml` | 各类别抓取参数和放置姿态 |
| `config/stacking.yaml` | 色块堆叠层位配置 |
| `config/categories.yaml` | 识别类别到库区的映射 |
| `config/web.yaml` | Web 端口、视频源、存储和 Mock 开关 |
| `config/calibration.md` | 标定文件和参数说明 |

调参优先修改 `config/*.yaml`，不要直接在代码里写死参数。

## 7. 数据集和模型流程

### 7.1 数据采集

```bash
python scripts/dataset/collect_images.py
```

采集后检查图片质量，确保不同光照、角度和背景都有覆盖。

### 7.2 标注校验和划分

```bash
python scripts/dataset/validate_labels.py
python scripts/dataset/split_dataset.py
python scripts/dataset/gen_dataset_yaml.py
```

### 7.3 模型训练、导出和转换

```bash
python scripts/model/train_yolov5.py
```

导出 ONNX 参考：

```text
scripts/model/export_onnx.md
```

转换 Atlas `.om` 模型：

```bash
bash scripts/model/convert_om.sh
```

推理测试：

```bash
python scripts/model/infer_test.py
```

## 8. 故障排查

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| Web 页面打不开 | 后端未启动或端口不对 | 检查 `uvicorn` 进程和端口 8000 |
| 实时画面黑屏 | 摄像头被占用或设备号错误 | 关闭多余浏览器标签，检查 `/dev/video0` |
| 点击抓取后失败 | ROS2 逆解服务未启动 | 先启动 `trial_service` 或 `dofbot_server` |
| 抓取位置偏移 | 标定或 offset 不准 | 重新标定，调整 `config/offset.txt` |
| 逆解失败 | 目标超出机械臂可达范围 | 调整物体位置或 `config/arm.yaml` 可达范围 |
| 夹爪夹不稳 | 夹爪角度或下降高度不合适 | 调整 `config/grasp.yaml` |
| 识别框抖动 | 光照变化或模型数据不足 | 固定光源，补充数据重新训练 |
| 库存不更新 | Runner 未启动或 Web 状态未同步 | 检查后台日志和 Web 后端 |

## 9. 答辩演示建议流程

1. 展示项目目录和系统架构，说明 PC 逻辑层、真机层、Web 层分工。

3. 运行 `python scripts/demo_full_system.py`，展示完整流程。
4. 打开 Web 页面，演示实时画面、状态、库存和日志。
5. 演示颜色方块抓取或堆叠。
6. 演示电力物品分类抓取。
7. 说明真机依赖：标定文件、`.om` 模型、ROS2 逆解、机械臂串口驱动。

## 10. 上传和归档注意事项

上传 GitHub 前建议保留源码、配置、文档和必要图片；忽略缓存、临时文件、运行日志和超大生成文件。

建议忽略：

- `__pycache__/`
- `.pytest_cache/`
- `.robotarm_tmp/`
- `web/backend/data/`
- `logs/`
- `models/*.om`
- `dataset/raw/`
- `dataset/labeled/`
- `config/dp.bin`
- `config/XYT_config.txt`
- `config/offset.txt`

GitHub 普通仓库单文件限制为 100MB。如果 PPT、视频、模型等文件超过 100MB，需要删除、压缩到 100MB 以下，或改用 Git LFS。
