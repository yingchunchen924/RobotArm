# RobotArm 前端操作手册（网页点按钮操作）

本文档讲：**启动服务后，怎么打开网页、各个按钮干什么、按你的实际流程一步步操作。**
适用于不想敲命令、全程在网页上点按钮的场景。命令行手动跑见文末附录。

## 一、操作前：先启动三样东西

在 MobaXterm 里连上开发板后，**按顺序**确认/启动：

### 1. IK 逆解服务（trial_service）—— 抓取的前提
先查在不在跑：
```bash
ros2 service list | grep trial_service
```
- 有输出 `/trial_service` → 已在跑，跳过。
- 没输出 → **新开一个 MobaXterm 标签页**启动它（它占前台，别关这个窗口）：
  ```bash
  bash /home/HwHiAiUser/start_ik.sh
  ```
  看到 `ROS Server Starts..` 即成功，窗口保持开着。

> ⚠️ IK 服务没起，点任何抓取按钮都会失败（前端显示 FAILED）。

### 2. 机械臂电源
确认机械臂扩展板通电（蓝灯亮）、摄像头插在 `/dev/video0`。

### 3. 网页后端
```bash
bash /home/HwHiAiUser/RobotArm/start_web.sh
```
（脚本会自动杀旧进程再起，日志在 `/tmp/web.log`。）

### 4. 打开网页
浏览器访问：**http://192.168.137.100:8000/**
- 右上角应显示「**真机模式**」。
- 「实时画面」应能看到摄像头画面。
- 如果画面黑：只留**一个**浏览器标签页（多标签会抢摄像头），Ctrl+F5 强刷。

## 三、「机械臂控制」面板（颜色方块）

用于 **YAHBOOM 垃圾分类垫上的红/黄/蓝/绿方块**。靠摄像头 HSV 颜色识别。

| 按钮 | 作用 |
|---|---|
| **抓红色 / 抓黄色 / 抓蓝色 / 抓绿色** | 只抓该颜色方块一次，放到对应库区（红=有害、黄=其他、蓝=可回收、绿=厨余） |
| **＋红/＋黄/＋蓝/＋绿**（自定义顺序区） | 点一下加入抓取队列，可重复、可只选几个 |
| **按此顺序抓** | 按你刚点的「已选顺序」依次连贯抓取 |
| **清空顺序** | 清掉自定义顺序 |
| **连贯全抓（红→黄→蓝→绿）** | 固定顺序把四色依次抓完 |
| **堆叠（红→黄→蓝→绿码4层）** | 把四色方块依次抓取、码放到中心十字，堆 4 层 |
| **停止** | 中断当前抓取/堆叠 |
| **回到初始姿态** | 机械臂回 home |
| **清空统计** | 库房统计、日志清零 |

### 典型流程（颜色分类）
1. 把红/黄/蓝/绿方块放到垫子各自取料区。
2. 点 **连贯全抓**（或单色按钮，或自定义顺序）。
3. 看「当前流程」依次走完，库房统计 + 操作日志自动更新。

### 典型流程（堆叠）
1. 四色方块放到各自取料区（红方块放**左上取料区**）。
2. 点 **堆叠**。
3. 机械臂按 红→黄→蓝→绿 依次抓取、码到中心十字。
> 注：堆叠运行时机械臂会自己转去看/抓每个方块，实时画面会跟着动。

## 四、「实物分类（点照片抓取）」面板（真实物件）

用于**铜端子、铜管、U盘、电池**等真实物件（没有颜色识别，靠你在照片上点位置）。

### 操作步骤
1. 点 **拍工作台照片**：机械臂回拍照位拍一张静态照片，显示在下方（拍照时实时画面会短暂暂停，正常）。
2. 点一个**物体按钮**选目标（决定放到哪个区）：
   | 按钮 | 物体 | 放置区 |
   |---|---|---|
   | 铜端子→蓝 | 铜端子 | 蓝区（可回收） |
   | 铜管→蓝 | 铜管 | 蓝区（可回收） |
   | U盘→灰 | U盘 | 灰区（其他） |
   | 电池→红 | 电池 | 红区（有害，自动下压抓取） |
3. 选模式：
   - **单次抓取**：在照片上点物体中心 → 立刻抓这一个。
   - **连贯队列**：依次「选物体 → 点照片位置」加入队列，最后点 **连贯执行队列** 一起跑。
4. **点击位置**会显示你点的像素坐标，**抓取队列**显示已排的项。
5. **清空队列** 清掉队列。

### 关键技巧（抓得稳）
- **点物体的圆柱/中心实心部位**，别点端口、边缘、带孔的环。
- 物体之间**别摆太紧凑**，留出夹爪下去的空间。
- 物体别放在**画面最远端边角**（机械臂极限位，难抓）。
- 每抓一个最好**重新拍照**再点下一个（抓取会碰动旁边物体）。

## 五、常见问题

| 现象 | 原因 / 处理 |
|---|---|
| 点按钮显示 FAILED | IK 服务（trial_service）没起，按第一节启动 |
| 实时画面黑屏 | 只留一个浏览器标签页，Ctrl+F5 强刷 |
| 拍照按钮无反应/提示忙 | 正在抓取，等抓完再拍 |
| 抓取时画面暂停 | 正常，摄像头让位给抓取，抓完恢复 |
| 实物抓取夹空 | 点物体中心实心部位；电池等矮物已自动下压 |

## 附录：手动命令行跑（不用网页）

```bash
# 1. 确认 IK（同上）
ros2 service list | grep trial_service

# 2. 颜色全抓 / 单色 / 堆叠（脚本已自动 source ROS 环境）
bash /home/HwHiAiUser/RobotArm/run_robotarm_env.sh auto --execute            # 颜色全抓
bash /home/HwHiAiUser/RobotArm/run_robotarm_env.sh auto --execute --color red # 只抓红
bash /home/HwHiAiUser/RobotArm/run_stacking_env.sh --execute                  # 堆叠

# 实物按坐标抓（CX CY 为 640x480 画面像素，ZONE=red/yellow/blue/green）
source /opt/ros/humble/setup.bash
source /home/HwHiAiUser/E2ESamples/src/E2E-Sample/ros2_robot_arm/ros2_ws/install/setup.bash
cd /home/HwHiAiUser/RobotArm
python3 scripts/board/pick_xy.py 320 240 blue --execute --close 180           # 铜件/U盘
python3 scripts/board/pick_xy.py 500 170 red  --execute --close 180 --grip-down -14  # 电池（下压）
```
> 手动跑前若网页开着、抢摄像头，先 `pkill -f uvicorn` 停网页；跑完 `bash .../start_web.sh` 恢复。
> 不带 `--execute` = 空跑只打印计划，机械臂不动，可先验证。
