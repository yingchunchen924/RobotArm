# 基于华为 Atlas 200I DK 智能机械臂的电力智能运维/库房管理系统开发计划

## 1. 项目背景

本项目对应实践题目 8：基于华为 Atlas 200I DK 智能机械臂的电力智能运维/电力库房管理。实验要求包含：

- 智能机械臂安装部署
- 目标识别
- 使用机械臂实现目标识别与抓取
- 电子元器件、电力金具或工具识别与抓取
- 色块识别、抓取与堆叠
- 可视化电力智能运维系统或电力库房管理系统

参考文档为《Atlas 200I DK A2 开发者套件 机械臂应用开发指南》，文档中的基础样例包含任意位置积木分拣、任意位置积木堆叠、摄像头标定、YOLOv5 OM 模型推理、ROS2 运动学服务和机械臂串口控制。本项目将在该样例基础上扩展为面向电力场景的识别、抓取、分类、移动和可视化管理系统。

## 2. 使用的编程语言与技术栈

| 语言 | 用途 |

## 3. 总体目标

实现一个完整的电力库房智能识别与机械臂抓取系统：

1. 摄像头实时采集标定板区域图像。
2. 系统识别色块、电子元器件、电力金具或工具。
3. 将图像坐标转换为机械臂基坐标。
4. 通过 ROS2 逆运动学服务求出机械臂关节角。
5. 控制机械臂完成抓取、移动、分类放置或堆叠。
6. 在可视化界面展示实时画面、识别类别、置信度、位置、抓取状态和库房统计。

## 4. 参考文档关键结论

### 4.1 硬件安装部署

参考文档第 2 章，安装部署流程如下：

1. PC、路由器和 Atlas 开发板接入同一网络。
2. 开发板通过 RJ45 网线连接网络。
3. 使用 SD 卡烧录 Ubuntu OS 镜像。
4. 通过 40PIN 排线连接 Atlas 开发板和机械臂扩展板。
5. USB 摄像头连接开发板 USB 接口，并固定在机械臂或支架上。
6. 开发板和机械臂分别接通电源，机械臂扩展板开关拨到 `ON`。
7. 摄像头对准带标定框的地图区域。

### 4.2 环境准备

参考文档第 3 章，需要完成：

1. 下载机械臂样例代码。
2. 安装 ROS2 Humble、colcon、pip。
3. 安装 ais_bench 和 aclruntime。
4. 安装 `requirements.txt` 中的 Python 依赖。
5. 安装机械臂驱动。
6. 编译 ROS2 工作空间。
7. 配置 `LD_LIBRARY_PATH`，复制 `libdofbot_kinemarics.so` 到系统库目录。

### 4.3 摄像头标定

参考文档第 3.3 节，首次使用需要运行 `相机校准.ipynb`：

1. 打开 Jupyter Notebook。
2. 进入机械臂样例的 `tools` 目录。
3. 运行相机校准 notebook。
4. 调整 `joint1` 和 `joint2`，使蓝色边框覆盖整个十字标定框。
5. 点击 `calibration_ok`，生成或更新标定配置。

关键配置文件：

- `dp.bin`：透视变换矩阵
- `XYT_config.txt`：相机标定配置
- `offset.txt`：机械臂硬件误差补偿

### 4.4 样例代码结构

参考文档第 4 章，原始样例主要包含以下功能包：

| 功能包 | 作用 |
| --- | --- |
| `dofbot_info` | 定义 ROS2 服务消息 `Kinemarics.srv` |
| `dofbot_moveit` | 提供机械臂 URDF、三维模型、正逆运动学服务 |
| `dofbot_garbage_yolov5` | 色块/目标识别与分拣逻辑 |
| `dofbot_color_stacking` | 色块识别与堆叠逻辑 |

### 4.5 机械臂控制核心流程

参考文档中的抓取流程可以抽象为：

1. 摄像头读取图像。
2. 图像缩放到 `640 x 480`。
3. 根据 `dp.bin` 进行透视变换。
4. YOLOv5 或颜色识别获得目标类别和中心点。
5. 将像素坐标转换为机械臂工作平面坐标。
6. 调用 ROS2 `Kinemarics` 服务，使用 `ik` 模式求逆运动学。
7. 机械臂移动到目标上方。
8. 松开夹爪。
9. 下降到目标位置。
10. 夹紧夹爪。
11. 抬起。
12. 移动到目标分类区或堆叠区。
13. 松开夹爪完成放置。

## 7. 开发阶段计划

### 阶段一：硬件安装与环境复现

目标：确保机械臂、摄像头、Atlas 开发板和 ROS2 样例能正常运行。

任务：

1. 按参考文档完成硬件连接。
2. 登录 Atlas 开发板。
3. 下载 `ros2_robot_arm` 样例代码。
4. 安装 ROS2、ais_bench、aclruntime、OpenCV 等依赖。
5. 编译 ROS2 工作空间。
6. 启动运动学服务：

```bash
source setenv.sh
ros2 run dofbot_moveit dofbot_server
```

7. 启动色块分拣样例：

```bash
source setenv.sh
ros2 run dofbot_garbage_yolov5 block_cls
```

8. 启动色块堆叠样例：

```bash
source setenv.sh
ros2 run robot_arm_color_stacking color_stacking
```

产出：

- 环境部署记录
- 样例运行截图或视频
- 机械臂能完成基础色块抓取

验收标准：

- 摄像头能正常读取画面。
- ROS2 服务能启动。
- 机械臂能执行至少一次完整抓取动作。

### 阶段二：摄像头标定与坐标转换

目标：建立稳定的像素坐标到机械臂坐标的转换关系。

任务：

1. 运行相机校准 notebook。
2. 调整摄像头角度和机械臂初始姿态。
3. 生成并保存 `dp.bin`、`XYT_config.txt`、`offset.txt`。
4. 编写坐标转换测试脚本，输入图像中心点，输出机械臂坐标。
5. 对标定板不同位置进行抓取测试。
6. 根据偏差调整 `offset.txt`。

产出：

- 标定配置文件
- 坐标转换测试脚本
- 偏差记录表

验收标准：

- 目标中心点识别稳定。
- 机械臂抓取误差控制在可接受范围内。
- 同一位置重复抓取成功率较高。

### 阶段三：色块识别、抓取与堆叠

目标：完成题目要求中的色块识别、抓取与堆叠。

任务：

1. 复用参考文档中的 `dofbot_color_stacking` 功能包。
2. 梳理 `main.py`、`stacking_target.py`、`stacking_grap.py` 的执行流程。
3. 将堆叠目标位置配置化，避免硬编码散落在代码中。
4. 支持至少 3 到 4 次连续堆叠。
5. 增加执行状态输出：等待识别、已识别、抓取中、放置中、完成、失败。

产出：

- 色块堆叠功能
- 堆叠动作参数配置
- 运行状态日志

验收标准：

- 能识别不同颜色色块。
- 能抓取指定色块并移动到堆叠区。
- 能完成多层堆叠演示。

### 阶段四：电力物品识别数据集准备

目标：准备电子元器件、电力金具或工具的数据集。

建议识别类别：

| 类别 | 示例 |
| --- | --- |
| 电子元器件 | 电阻、电容、继电器、接线端子、开关模块 |
| 电力金具 | 线夹、螺栓、垫片、挂环、绝缘子小件 |
| 工具 | 螺丝刀、扳手、钳子、卷尺、电工刀 |

任务：

1. 确定最终识别类别，建议先选 3 到 5 类。
2. 每类采集 100 到 300 张图片。
3. 保证不同角度、不同光照、不同摆放位置。
4. 使用 LabelImg、Roboflow 或 CVAT 进行 YOLO 格式标注。
5. 划分训练集、验证集和测试集。
6. 编写数据集配置 `power_objects.yaml`。

产出：

- 原始图片
- 标注文件
- 数据集配置文件
- 类别表

验收标准：

- 每个目标都有准确标注框。
- 数据集能被 YOLO 训练脚本正常读取。

### 阶段五：电力物品目标识别模型

目标：训练或适配电力物品识别模型，并转换为 Atlas 可运行的 `.om` 模型。

任务：

1. 使用 YOLOv5 训练电力物品检测模型。
2. 导出 ONNX 模型。
3. 使用 Atlas 模型转换工具转换为 `.om`。
4. 将 `.om` 模型上传到开发板。
5. 修改推理配置和类别文件 `coco_names.txt` 或自定义 `power_names.txt`。
6. 在开发板上进行静态图片推理测试。
7. 在摄像头实时画面中测试推理速度和识别准确率。

产出：

- `power_objects.om`
- `power_names.txt`
- 推理测试脚本
- 识别准确率记录

验收标准：

- 摄像头画面中能框出目标物品。
- 能输出类别、置信度和中心点坐标。
- 识别结果能传递给机械臂控制模块。

### 阶段六：电力物品抓取与分类移动

目标：实现电子元器件、电力金具或工具的识别、抓取与移动。

任务：

1. 新建 `power_object_detect` 或改造原 `dofbot_garbage_yolov5` 功能包。
2. 将目标类别映射到库房区域，例如：

| 识别类别 | 放置区域 |
| --- | --- |
| resistor | 电子元器件区 |
| capacitor | 电子元器件区 |
| wrench | 工具区 |
| screwdriver | 工具区 |
| clamp | 电力金具区 |

3. 编写目标筛选策略：
   - 优先抓取置信度最高的目标。
   - 如果多个目标重叠，选择离机械臂最近或最容易抓取的目标。
   - 如果目标在不可达区域，跳过并提示。
4. 复用 ROS2 `Kinemarics` 服务计算逆运动学。
5. 为不同物体设置夹爪角度和下降高度。
6. 增加异常处理：
   - 未识别到目标
   - 逆解失败
   - 抓取后物体滑落
   - 摄像头读取失败

产出：

- 电力物品识别抓取脚本
- 类别到库位的映射配置
- 抓取参数配置

验收标准：

- 至少能识别并抓取 3 类电力相关物品。
- 能按类别移动到指定区域。
- 每次操作都有状态输出。

### 阶段七：可视化电力库房管理系统

目标：完成题目要求中的可视化系统。

推荐功能：

1. 实时摄像头画面显示。
2. 识别框、类别和置信度显示。
3. 当前机械臂状态显示。
4. 库房区域统计：
   - 电子元器件数量
   - 电力金具数量
   - 工具数量
   - 未分类数量
5. 操作日志：
   - 时间
   - 识别类别
   - 置信度
   - 抓取结果
   - 放置区域
6. 手动控制按钮：
   - 开始识别
   - 开始抓取
   - 暂停
   - 回到初始姿态
   - 清空统计

后端接口建议：

| 接口 | 方法 | 作用 |
| --- | --- | --- |
| `/api/status` | GET | 获取系统状态 |
| `/api/detections` | GET | 获取最近识别结果 |
| `/api/tasks/start` | POST | 开始自动识别抓取 |
| `/api/tasks/stop` | POST | 停止任务 |
| `/api/arm/home` | POST | 机械臂回到初始姿态 |
| `/api/inventory` | GET | 获取库房统计 |
| `/video` | GET | 获取实时视频流 |

产出：

- Web 后端服务
- Web 前端页面
- 识别与抓取日志
- 库房状态数据

验收标准：

- 浏览器能打开管理页面。
- 页面能显示实时画面和识别结果。
- 页面能显示机械臂状态和操作记录。
- 页面能触发开始、停止和复位动作。

### 阶段八：系统联调与最终演示

目标：把机械臂、AI 识别、抓取控制和可视化系统整合为完整演示。

任务：

1. 编写一键启动脚本。
2. 启动 ROS2 运动学服务。
3. 启动识别抓取主程序。
4. 启动可视化后端和前端。
5. 进行连续识别抓取测试。
6. 记录每次识别和抓取结果。
7. 拍摄最终演示视频。

建议演示流程：

1. 展示硬件连接和系统首页。
2. 放置色块，演示色块识别和堆叠。
3. 放置电力工具或元器件，演示识别抓取。
4. 系统页面展示类别、置信度和库房统计变化。
5. 机械臂回到初始姿态。

产出：

- 一键启动脚本
- 最终演示视频
- 实验报告材料
- 测试记录

验收标准：

- 系统能稳定完成完整闭环。
- 可视化界面能实时反映识别和抓取结果。
- 满足实践内容 6 项要求。

## 8. 关键模块设计

### 8.1 图像采集模块

职责：

- 打开 USB 摄像头。
- 读取每一帧图像。
- 统一缩放为 `640 x 480`。
- 输出给标定和识别模块。

推荐语言：Python。

### 8.2 标定与透视变换模块

职责：

- 读取 `dp.bin`。
- 对画面进行透视变换。
- 将标定板区域转为标准平面。

推荐语言：Python + OpenCV。

### 8.3 目标识别模块

职责：

- 加载 `.om` 模型。
- 对图像进行推理。
- 输出目标类别、置信度、检测框、中心点。

推荐语言：Python。

### 8.4 坐标转换模块

职责：

- 将像素中心点转换为机械臂平面坐标。
- 加入 `offset.txt` 硬件误差补偿。
- 判断目标是否在机械臂可达范围内。

推荐语言：Python。

### 8.5 运动学服务模块

职责：

- 接收目标坐标。
- 使用 `ik` 模式求逆运动学。
- 返回机械臂各关节角度。

推荐语言：C++，复用参考文档中的 ROS2 服务。

### 8.6 机械臂执行模块

职责：

- 控制机械臂移动到目标上方。
- 控制夹爪开合。
- 控制机械臂移动到分类区或堆叠区。
- 处理执行过程中的状态和异常。

推荐语言：Python。

### 8.7 可视化系统模块

职责：

- 展示实时视频流。
- 展示识别结果。
- 展示机械臂状态。
- 展示库房统计。
- 提供手动控制按钮。

推荐语言：Python + HTML/CSS/JavaScript。

## 12. 风险与解决方案

| 风险 | 表现 | 解决方案 |
| --- | --- | --- |
| 摄像头标定不准 | 抓取位置偏移 | 重新校准，调整 `offset.txt` |
| 光照变化影响识别 | 检测框不稳定 | 固定光源，增加不同光照数据 |
| 模型识别准确率低 | 误识别或漏识别 | 增加数据集，重新训练 |
| 机械臂逆解失败 | 无法到达目标 | 限制可抓取区域，跳过边缘目标 |
| 夹爪抓不稳 | 物体滑落 | 调整夹爪角度、下降高度和抓取速度 |
| Web 与机械臂主程序冲突 | 状态不同步 | 使用统一状态文件、消息队列或后端全局状态管理 |

## 13. 后续开发建议

1. 先完整复现参考文档样例，不要一开始就改复杂模型。
2. 将标定、识别、坐标转换、抓取动作分别写成独立模块。
3. 所有关节角、放置区域、类别映射都放到配置文件中。
4. 每次改动后只测试一个变量，例如只改 `offset` 或只改夹爪角度。
5. 先实现单个目标抓取，再做连续抓取和可视化。
6. 最终演示时优先选择形状规则、容易夹取、识别稳定的电力物品。
